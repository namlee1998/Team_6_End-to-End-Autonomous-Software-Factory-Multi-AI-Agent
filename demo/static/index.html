System Idea

Why this demo uses a LangGraph supervisor-worker model

Core Idea

This demo models an autonomous software factory where the Supervisor owns orchestration, not implementation. The Supervisor receives a feature request, creates a run trace, fans out work to specialized agents, then fans in the outputs for HITL review.

The important change is that PO, UX, DEV, and QA are not shown as a blind chain. Each worker has internal LangGraph stages, explicit input contracts, MCP/tool calls, A2A messages, and output artifacts.

Why Parallel Workers

  • Lower latency: independent analysis can run at the same time.
  • Better traceability: every worker has a parent run id and worker run id.
  • Targeted rework: HITL feedback routes only the owning worker.
  • Clear fault isolation: QA findings point back to UX, DEV, PO, or QA instead of restarting the whole chain.

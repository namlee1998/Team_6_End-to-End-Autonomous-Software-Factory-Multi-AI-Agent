Báo cáo sơ đồ BMAD HITL Flow
Sequence Diagram cho workflow 4 session: Mary - John - Sally - Amelia
Mục tiêu

Giải thích cách BMAD dùng nhiều session agent để tự động hóa SDLC, trong đó artifact được pass qua file system và human review
nằm ở các checkpoint tự nhiên.

Bản chất HITL

HITL trong BMAD là inherent: user đọc artifact, trả lời câu hỏi, approve/reject phase transition. Không cần implement approval
service riêng ở mức tool-call.

Thông điệp
chính

Đây là human-governed multi-agent SDLC workflow: agent tự động hóa việc chuyên môn, human giữ quyền quyết định ở các điểm
chuyển phase và release.

Cách đọc nhanh
Đọc sequence diagram từ trái sang phải để biết actor/participant, và từ trên xuống dưới để biết thứ tự thời gian. Mũi tên biểu diễn message hoặc artifact handoff.
Các khối màu cam là điểm HITL: user review, approve, reject hoặc đưa thêm thông tin.
Luồng tổng quát: User -> Mary phân tích business -> John tạo PRD/stories -> Sally tạo UX spec -> Amelia lập sprint, code theo TDD và tạo QA evidence -> User
quyết định deploy hoặc mở sprint mới.

5. Diễn giải dùng trong báo cáo / thuyết trình
Sơ đồ này mô tả BMAD workflow dưới dạng 4 session agent tách biệt. User bắt đầu bằng một feature request. Mary, với vai trò Business Analyst, tạo product brief
và PRFAQ để làm rõ vấn đề, target user và constraints. Sau đó user đọc artifact và quyết định chuyển sang John hoặc yêu cầu Mary sửa.
John, với vai trò Product Manager, đọc artifact của Mary, phỏng vấn user để làm rõ requirement, rồi tạo PRD và epics/stories. John cũng chạy readiness check để
đảm bảo artifact đủ điều kiện trước khi chuyển sang design và development.
Sally đọc PRD và stories để tạo UX spec và wireframe spec. User review UX trước khi chuyển sang Amelia.
Amelia đọc toàn bộ artifact đã duyệt, lập sprint plan, rồi implement từng story theo TDD: RED viết failing tests, GREEN implement để tests pass, REFACTOR clean up.
Sau mỗi story, user review code và quyết định approve hoặc yêu cầu sửa. Cuối sprint, Amelia tạo E2E tests, test plan và retrospective. User quyết định deploy, hold,
rollback hoặc bắt đầu sprint mới.

6. Kết luận kiến trúc
Ý nghĩa

Giải thích

Multi-agent thật

Mary, John, Sally và Amelia đảm nhiệm vai trò khác nhau trong SDLC thay vì một agent làm tất cả.

Artifact-first

Mỗi phase tạo file đầu ra rõ ràng. File là contract giữa các session và là bằng chứng audit.

HITL tự nhiên

User review artifact và quyết định chuyển phase. Đây là HITL ở cấp workflow, không phải approve từng action nhỏ.

Tránh approve fatigue

Human chỉ approve các checkpoint quan trọng: PRD, UX, readiness, story completion và deploy decision.
